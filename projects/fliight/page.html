Fliight VR App & Desktop Client

I was commissioned to create a VR application for Fliight Technologies, capable of streaming video from a network source and streaming back user input to the desktop client.

The VR app was made with Unity 2022.3 using the OpenXR libraries, and the OpenXR Meta extension to ensure Quest 2 and Quest 3 compatibility for the target devices.

3D environment, passthrough, video streaming

Web browser, panel resizing

Desktop client, controller streaming

The purpose of the application is to serve as an FPV alternative for controlling drones and planning flight paths.


To achieve this I had to implement the following features:

  • - Video streaming through an RTSP protocol, works with both local network and online streams. The video player uses the libVLC library to ensure a wide range of compatibility with different streaming protocols.
  • - Configuration panel to handle connection settings.
  • - Joystick controller input streaming back to a desktop client on a separate computer. The client was implemented in WPF and uses ViGEmBus to emulate an Xbox360 controller. The streaming and connection protocol for user input is custom made, and is focused on low-latency.
  • - Web browser window to view websites and use google maps. Multiple browser windows can be opened at a time for maximum productivity.
  • - A custom 3D panel manipulator for grabbing and resizing the browser and video stream panels.

Most 2D Textures, 3D models and sound effects are licensed from third-party vendors, or are provided by Fliight Technologies.
